PART 1 — WHAT THIS PROJECT IS
JanSetu is a monorepo containing two web apps and multiple backend services:
App 1 — Citizen Portal (jansetu.in equivalent on Replit → runs on port 3000) Citizens file complaints in any Indian language, track status, give feedback.
App 2 — Officer Portal (officer.jansetu.in equivalent → runs on port 3001) Government officers receive complaints, update status, resolve, escalate.
Backend API (FastAPI Python → port 8000) Single REST API serving both apps. JWT authentication. All business logic here.
AI Engine (FastAPI Python → port 8001) Separate service running all AI models: DistilBERT, MiniLM, CLIP, Whisper, XGBoost.
Realtime Service (Node.js + Socket.io → port 3002) Live updates: new complaints appear on officer dashboard without refresh.
Database (PostgreSQL via Supabase — already connected) Connection string already in environment variables.

PART 4 — LOGIN ISSUE ROOT CAUSES AND FIXES
The login/fetch not working has these possible causes. Check ALL of them:
Cause 1 — CORS not configured in backend
In services/backend/app/main.py, the CORS middleware must allow the frontend origins. If this is wrong, the browser blocks every API call silently with a CORS error.
Fix: Make sure this exists in main.py:
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://localhost:3000",
        "http://localhost:3001",
        "https://[YOUR-REPLIT-URL]",
        "*"  # For development only
    ],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
Cause 2 — Wrong API URL in frontend
The citizen/officer apps are calling the wrong backend URL. Check apps/citizen/app/(auth)/login/page.tsx or wherever the login API call is made.
The API call should look like:
POST {NEXT_PUBLIC_API_URL}/auth/login
Body: { phone: "9876543210", password: "Commander@123" }
If NEXT_PUBLIC_API_URL is undefined or wrong, all fetches will fail.
Fix: Add console.log(process.env.NEXT_PUBLIC_API_URL) temporarily to verify the value.
Cause 3 — Database connection string wrong
asyncpg driver requires postgresql+asyncpg:// not postgresql://
Fix: Make sure DATABASE_URL starts with: postgresql+asyncpg://
Cause 4 — Backend not running when frontend calls it
On Replit, all services must start before the frontend. If backend hasn't started yet, every fetch returns connection refused.
Fix: Add a startup health check. In the api-client, before making any call, verify the backend is reachable at /health endpoint.
Cause 5 — Password hash mismatch
The seed data used crypt() from pgcrypto. The backend login must verify using the same algorithm. In services/backend/app/routers/auth.py:
from passlib.context import CryptContext
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

# Verify: pwd_context.verify(plain_password, hashed_password)
# Hash:   pwd_context.hash(plain_password)
The pgcrypto crypt() and Python bcrypt ARE compatible if using bf (blowfish) salt. If they are not matching, reset the password with:
UPDATE users
SET password_hash = crypt('Commander@123', gen_salt('bf', 10))
WHERE phone = '9876543210';
Cause 6 — JWT token not being sent in subsequent requests
After login, the token must be stored and sent with every request. In packages/api-client, the Axios interceptor must add the header:
axiosInstance.interceptors.request.use((config) => {
  const token = localStorage.getItem('jansetu_token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});
If the token key name is different (e.g., 'token' vs 'jansetu_token'), login succeeds but all authenticated routes return 401 Unauthorized.
Cause 7 — Next.js 14 Server vs Client component fetch
In Next.js 14 App Router, components that use localStorage, hooks, or browser APIs MUST have 'use client' at the top. If login page is a Server Component trying to call client-side APIs, it will fail.
Fix: Add 'use client' to the top of every page that:
* Uses useState, useEffect
* Makes API calls on user interaction
* Reads localStorage
* Uses Socket.io

Create start.sh in root:
#!/bin/bash
echo "Starting JanSetu services..."

# Install dependencies if not installed
cd services/backend && pip install -r requirements.txt -q &
cd services/ai-engine && pip install -r requirements.txt -q &
cd services/realtime && npm install -q &
cd apps/citizen && npm install -q &
cd apps/officer && npm install -q &
wait

echo "Dependencies installed. Starting services..."

# Start backend
cd services/backend
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload &
BACKEND_PID=$!
echo "Backend started (PID: $BACKEND_PID)"

# Wait for backend to be ready
sleep 5

# Start AI engine (with fallback if models not loaded)
cd ../ai-engine
uvicorn main:app --host 0.0.0.0 --port 8001 --reload &
AI_PID=$!
echo "AI Engine started (PID: $AI_PID)"

# Start realtime service
cd ../realtime
node server.js &
RT_PID=$!
echo "Realtime started (PID: $RT_PID)"

# Start citizen app
cd ../../apps/citizen
npm run dev -- --port 3000 &
CITIZEN_PID=$!
echo "Citizen app started (PID: $CITIZEN_PID)"

# Start officer app
cd ../officer
npm run dev -- --port 3001 &
OFFICER_PID=$!
echo "Officer app started (PID: $OFFICER_PID)"

echo "All services running!"
echo "Citizen: http://localhost:3000"
echo "Officer: http://localhost:3001"
echo "API:     http://localhost:8000/docs"
echo "AI:      http://localhost:8001/docs"

# Keep alive
wait

PART 12 — COMMON REPLIT-SPECIFIC ISSUES AND FIXES
Issue: "Module not found" in Next.js
Fix: Make sure packages/ui, packages/types, packages/api-client are properly linked in apps/citizen/package.json and apps/officer/package.json as workspace dependencies.
Issue: Port already in use
Fix: In .replit or start.sh, kill existing processes before starting:
pkill -f uvicorn || true
pkill -f "next dev" || true
pkill -f "node server" || true
Issue: Replit URL changes, breaking API calls
Fix: Use relative API routes in Next.js instead of absolute URLs. In apps/citizen/app/api/[...path]/route.ts, proxy all /api calls to backend. This way the frontend always calls /api/* relative to itself.
Issue: Database SSL required by Supabase
Fix: In DATABASE_URL append ?sslmode=require:
postgresql+asyncpg://postgres:pass@db.xxx.supabase.co:5432/postgres?ssl=true
In SQLAlchemy engine creation:
engine = create_async_engine(
    DATABASE_URL,
    connect_args={"ssl": True}
)
Issue: Slow startup on Replit free tier
Fix: The AI engine takes longest to start. Add a loading state to both frontends that polls /api/v1/health every 2 seconds and only shows the app when backend responds.
